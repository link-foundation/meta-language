## Notes on scope decisions

- **React vs. plain JS (R-5):** the issue permits React but does not require it.
  A static, dependency-free landing page is simpler to build and deploy on Pages
  and keeps the demo's only heavy dependency (the wasm bundle) deliberate. The
  interactive logic is pure Rust compiled to wasm, satisfying the "Rust (wasm)"
  intent directly.
- **Why the demo wraps `links-notation`, not full `meta-language` (R-3):** the
  full crate links native tree-sitter C grammars (and `lindera`/`lingua`) which
  cannot target `wasm32-unknown-unknown`. `links-notation` is pure Rust and is
  the structural substrate the meta language serializes to, so it is the correct
  wasm-compatible surface for a live demo. Richer language features are shown via
  pre-rendered examples on the landing page.
